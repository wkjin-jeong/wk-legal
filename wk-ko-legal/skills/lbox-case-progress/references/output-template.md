`$LBOX_DIR/lbox-case-progress.md` (기본 `~/Library/CloudStorage/Dropbox/AI/lbox-case-progress/lbox-case-progress.md`) 에 작성되는 통합 보고서의 정해진 구조다.

## 양식 규칙

### 메타 상단

- "최종 갱신" 은 KST 기준 `YYYY-MM-DD HH:MM` 형태.
- "직전 갱신" 은 가장 최근 스냅샷의 timestamp. 없으면 줄 자체를 생략.
- "변경" 한 줄은 직전 스냅샷이 없으면 생략.
- 사건 수는 `등록 N건 (진행 X · 종결 Y)` 식으로 합산.

### 변경 요약 테이블

- 변경된 사건만 (신규 사건, 신규 이벤트가 있는 사건, 신규 종결 사건) 표시. 변경 없으면 본 섹션 자체를 생략하고 한 줄 텍스트(`변경 사항 없음`)로 대체.
- 동일 사건에 여러 변경이 있으면 한 행에 세미콜론으로 합친다.

### 진행 중 / 종결 사건 표

- **다음 기일** 칸은 `YYYY-MM-DD HH:MM 종류` 형태. 다음 기일이 없으면 빈 칸.
- **최종 갱신** 은 `events.updated_date` 를 YYYY-MM-DD 로.
- 행은 다음 기일 임박 순(오름차순), 다음 기일이 없는 사건은 맨 아래로.
- 종결 사건은 진행 사건과 분리해 별도 표.

### 사건별 상세

- 사건 번호 순(통합 표의 # 그대로) 으로 나열.
- 진행이력 표는 **최신 → 과거** 순(내림차순).
- `progress.type` 매핑:
  - `date` 또는 `hearing` → "기일"
  - `submit` → "제출서류"
  - `send` → "송달"
  - `order` → "명령"
  - `doc` → "문서"
  - `etc` 또는 기타 → "기타"
  - (그 외 알 수 없는 값은 원문 그대로)
- `forCalendar=true && type∈{date,hearing}` 인 항목은 행 좌측에 `★` 표시.
- `scheduleModified=true` 인 기일은 결과 칸에 `(기일변경)` 추가.
- 일자가 빈 문자열/null 인 항목은 "(일자 미상)" 로 표기.

### 에러 사건

- `errorFlag=true` 인 사건은 상세 섹션 헤더에 ` ⚠️ lbox 조회 실패` 를 붙이고, 이벤트 표 대신 한 줄로:
  > "lbox 가 법원 시스템에서 이 사건 정보를 받아오지 못했습니다. lbox 홈페이지에서 직접 확인해 주세요."

### 빈 보고서

사건이 0건이면 상단 메타 다음에 한 단락만 둔다:
> "현재 lbox 사건일정관리에 등록된 사건이 없습니다."

## 변경 감지 규칙

`diff_report.py` 가 출력하는 `diff.json` 구조:

```json
{
  "previousAt": "2026-05-13T08:30:00+09:00",
  "currentAt": "2026-05-14T13:42:00+09:00",
  "newCases": [ {"id": 356877, "caseNo": "...", "caseName": "..."} ],
  "closedCases": [ {"id": ..., "caseNo": "...", "caseName": "..."} ],
  "newEvents": {
    "356880": [
      {"dateMs": ..., "type": "send", "content": "답변서 송달", ...}
    ]
  },
  "scheduleChanges": {
    "356881": "다음 기일 2026-06-09 14:00 → 2026-06-23 10:00"
  }
}
```

- `newCases`: 직전 스냅샷에 없던 id 가 현재에 있음.
- `closedCases`: 직전 스냅샷에는 있으나 현재 응답에 없음(삭제·종결 모두 포함).
- `newEvents`: 사건의 `events` 배열을 `(dateMs, type, content)` 키로 비교해 신규만.
- `scheduleChanges`: `nextEvent.dateMs` 가 변경되었거나 `scheduleModified=true` 인 신규 항목.
```
