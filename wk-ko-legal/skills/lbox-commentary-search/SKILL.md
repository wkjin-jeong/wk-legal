# lbox-commentary-search

lbox.kr 주석서·실무서 검색에서 사용자 질의와 관련된 주석서 섹션을 자동으로 검색하고, 본문을 조회하여 사용자 질의에 답하는 markdown 보고서를 생성합니다.

이 Skill은 `lbox-case-search`(판례 검색)의 자매 Skill이며, 같은 lbox.kr 개편판 Chrome 자동화 패턴을 따릅니다.

> **2026.6 사이트 전면 개편 반영본.** 개편 전 방식은 더 이상 동작하지 않습니다. 핵심 변화: ① 검색이 직접 URL(`/v2/search/commentary?...`)이 아니라 홈 **컴포저 "검색" 모드 → 작업(Task) → 결과 패널의 "주석·실무서" 탭**으로 바뀜, ② 숫자 `statute` 코드·`document=COMMENTARY/PRACTICAL` 파라미터 폐지 → **법령 이름·문서유형 필터**(`/book/list` 또는 패널 필터)로 대체, ③ 본문 viewer URL이 `/book/{bookId}?tocId=…&nodeId=…&volumeHistoryId=…` 형식, ④ 본문 페이지가 **2벌로 렌더**되어 dedup이 필요. 변경점 요약은 문서 끝 표를 참고하세요.

## 사전 조건 및 환경 점검

lbox.kr 로그인 + 주석·실무서 **구독이 항상 활성**이라고 전제합니다(구독 확인 분기를 두지 않음). 환경 점검은 `lbox-case-search`와 동일합니다.

1. **Chrome MCP 도구**: `mcp__Claude_in_Chrome__*`(`navigate`, `javascript_tool`, `computer`, `read_page` 등). 컴포저 입력·모드 선택·탭 클릭에는 `computer`/`find`가 필요합니다.
2. **연결된 브라우저 확인**: `list_connected_browsers` → 여러 기기면 `select_browser(deviceId)`. 대화마다 `tabs_context_mcp{createIfEmpty:true}`로 전용 탭 확보.
3. **검색은 '작업(Task)'을 생성**합니다(새 lbox의 정상 동작, 등록 '사건'과 별개). 한 검색 세션은 가급적 하나의 작업에서 진행하세요.
4. **JS 사전 점검**: 첫 페이지에서 `document.title` 평가로 정상 응답 확인.

## 작업 흐름

### 1단계: 사용자 의도 파악

- **검색어 (필수)**: 핵심 법률 쟁점·키워드. 자연어 질의는 짧은 키워드로 정제(예: "양도금지 특약이 있는 채권을 양도하면 효력이 있는지" → "양도금지특약").
- **법령 한정 (선택, 권장)**: 사용자가 법령을 명시하면 그 **이름**으로 좁힙니다(민법·민사소송법·상법·형법 등). 개편 후에는 숫자 코드가 아니라 법령 이름 필터를 씁니다. 사용 가능한 법령 목록·문서유형·필터 적용법은 `references/filter-map.md`를 이 시점에 읽습니다.
- **문서유형 (선택)**: 주석서 / 실무서 / 전체. 명시되지 않으면 전체에서 검색하고, 결과의 도서명으로 유형을 판별합니다(예: "민법 채권각칙 제N권" = 주석서, "…소송실무"·"…설명서" = 실무서).

질의가 모호하면 진행 전에 한 가지만 확인하세요.

### 2단계: 검색 실행 (컴포저 "검색" 모드 → 주석·실무서 탭)

1. `navigate` → `https://lbox.kr/`.
2. 컴포저 입력창에 검색어를 입력하고 **"검색"(돋보기) 모드**를 선택한 뒤 **제출(↑)**. (구체 조작은 `references/extraction.md` 1장. `lbox-case-search`와 동일.)
3. `/task/{id}`로 이동하며 결과 패널이 열립니다(기본 **판례** 탭).
4. 결과 패널 상단 탭에서 **"주석·실무서" 탭**을 클릭해 주석서 결과로 전환합니다. 이 탭은 **패널 안의 `role="tab"` 버튼**이며, **좌측 내비게이션의 "주석·실무서" 링크(`/book/list`로 이동)와 다릅니다** — 좌측 링크를 누르면 검색 결과를 잃고 도서 목록으로 가버리니 주의하세요(셀렉터 구분법은 `references/extraction.md` 1장).

> **검색·탭 전환 직후 짧게 대기**: 클릭과 추출 `javascript_tool` 호출을 분리하고 사이에 짧은 텀을 둡니다(timeout 회피).

법령·문서유형으로 좁히려면: (ⓐ) 주석·실무서 탭의 필터 칩(있으면 문서유형·법령)을 사용하거나, (ⓑ) 각 결과의 **도서명·breadcrumb**으로 결과 단계에서 선별하거나, (ⓒ) `https://lbox.kr/book/list`(문서유형·법령 필터가 있는 도서 목록)에서 대상 도서를 먼저 확인합니다. 자세한 건 `references/filter-map.md`.

### 3단계: 카드 추출 및 페이지 순회

주석·실무서 결과 카드는 `a[data-track-props]`(JSON `documentType:"scholar"`)이며, `docId`·`rank`와 **href에 본문 식별자**(bookId·nodeId·tocId·volumeHistoryId)가 들어 있습니다. 카드 컨테이너(`div.border-b-xs`)의 3부분(① 섹션 제목+도서메타, ② 스니펫, ③ breadcrumb)과 식별자 분해, 중복 제거 JS는 `references/extraction.md` 1장을 따릅니다.

#### 페이지 순회 정책

- 결과는 10건/페이지, 하단 **번호 페이지네이션**(판례 탭과 동일). **최소 2페이지(상위 20건)** 까지 수집하되, 누적 **High 후보 ≥ 5건**이면 본문 visit로 진입.
- 결과가 매우 적으면(예: 5건 미만) 추가 페이지 없이 곧바로 본문 visit. **0건**이면 키워드를 넓히거나 법령 한정을 풀어 1차 재검색을 제안(진행 전 한 줄 안내).

### 4단계: 1차 관련성 분류 (snippet triage)

각 결과를 4개 신호로 분류합니다. (판례 검색과 동일한 정신.)

- **섹션 제목(`sectionTitle`)**: 질의 키워드 직접 포함 시 ↑.
- **스니펫(`snippet`)**: 질의 쟁점을 정면으로 다루면 High, 키워드만 있고 쟁점이 다르면 Low.
- **breadcrumb**: 도서 내 위치(예: "제3편 채권 / 제5장 불법행위"). 질의 분야 일치 시 ↑.
- **도서명(`book`)**: 권 단위 적합성(예: 채권양도 → "민법 채권총칙", 불법행위 손해 → "민법 채권각칙").

**참고문헌 모드 보정**: 스니펫이 논문·문헌 목록(`민사판례연구|민사법학|…연구\(\d{4}\)`, 따옴표 논문 제목+연도, 저자+책+출판사+연도 패턴이 2회 이상)으로 채워졌으면 스니펫 가중치를 낮추고 섹션 제목·breadcrumb·도서명에 비중을 둡니다. 이때는 스니펫만으로 Low 처리하지 말고 Medium으로 두어 본문 visit 후 재판단(false negative 회피).

본문 visit은 **High + Medium 후보만** 수행합니다.

### 5단계: 본문 visit 및 섹션 추출

각 후보의 식별자로 본문 URL을 구성해 `navigate`합니다.

```
https://lbox.kr/book/{bookId}?tocId={tocId}&nodeId={nodeId}&volumeHistoryId={volumeHistoryId}
```

(카드 href를 그대로 navigate해도 됩니다 — 단 JS 결과에 raw href를 담아 반환하면 `[BLOCKED]`가 나므로, 식별자는 **분해된 필드로** 받아 URL을 구성합니다. `references/extraction.md` 1장.)

섹션 추출 메커니즘(URL의 `nodeId`에 해당하는 `[data-node-id]` 헤더부터 다음 `[data-viewer-type="header"]` 전까지 형제 수집, **2벌 렌더 dedup**, 섹션이 큰 챕터일 수 있어 전문은 window 변수에 저장하고 **질의 키워드 인근 발췌**로 반환)은 `references/extraction.md` 2장을 이 시점에 읽고 그대로 사용합니다.

#### 5.1 본문이 빈약할 때 — 같은 조문/장의 하위 노드 추가

조문 본문 노드(`제449조 [채권의 양도성]` 등)는 조문 ①②③항 + 관련조문 + 참고문헌만 있어 빈약(노드 < 10개 또는 < 2,000자, 또는 조문 텍스트+`[참고문헌]`/`[관련조문]`뿐)할 수 있습니다. 핵심 법리(Ⅰ. 의의, Ⅱ. 성립요건 …)는 같은 조문/장의 하위 노드에 분산됩니다. 이 경우:

1. 같은 검색 결과에서 **같은 도서(`book`) + 같은 breadcrumb 부모**를 공유하는 다른 카드(같은 조문/장의 하위 노드)를 찾습니다.
2. 질의에 가장 가까운 1~2개를 추가로 본문 visit합니다.
3. 같은 결과에 적합한 하위 노드가 없으면 viewer의 도서 목차에서 인접 노드를 확인하거나 다른 출판사 주석서를 visit합니다.

조문 본문 노드만 보고 답변하면 "법리 설명 없음"의 빈약한 답변이 됩니다.

#### 5.2 본문 추출 실패 시

- `header not found`: nodeId가 viewer에서 안 잡힘(로드 미완·UI 변경). 한 번 retry 후 실패면 그 카드 건너뜀.
- 섹션 길이 0 또는 200자 미만: retry 후에도 실패면 건너뜀(구독은 활성 전제이므로 구독 안내 표시 안 함).

#### 5.3 추가 단서 (선택)

핵심 주석서 1~2개에 한해 인용 판례(`\d{4}[가-힣]+\d+`)·인용 조문(예: "민법 제449조")·인용 학설을 추가 추출해 "주석서가 인용한 핵심 판례·법령" 표에 정리하면 답변 품질이 올라갑니다.

### 6단계: 본문 기반 최종 관련성 분류

- **직접 관련(High)**: 질의 쟁점을 정면으로 다룬 섹션 → "정리된 답변"+"참고 주석서" 양쪽 사용.
- **간접 관련(Medium)**: 동일 분야 일반론·비교 가치 → "참고 주석서"에 한 줄 요지만.
- **관련성 낮음(Low)**: 키워드만 우연히 일치 → "제외 섹션" 목록에 메타데이터만.

### 7단계: 결과 markdown 파일 작성

- **위치**: 사용자가 선택한 작업 폴더(없으면 outputs/ + `computer://` 안내). 절대 경로 하드코딩 금지.
- **파일명**: `lbox-commentary-<주요키워드>-<YYYYMMDD>.md`. 예) `lbox-commentary-양도금지특약-20260629.md`. 민감정보 제외.

보고서 표준 양식은 **`references/report-format.md`를 작성 직전에 읽고 그대로 따릅니다.**

#### 인용·발췌 가이드

lbox.kr은 구독자에게 본문 직접 인용을 허용합니다. 분량을 임의 제한하지 않되 가독성을 위해: **핵심 부분은 직접 인용(`"..."`)**, 그 외는 요약, **출처는 도서명·저자(편집대표)·판본까지 표기**(예: `[김용덕 편, 민법 채권각칙 제6권, 한국사법행정학회 제5판, 2021]`), **본문에서 확인되지 않은 내용 금지**.

#### 파일 안내

저장 후 경로 안내: Cowork/데스크톱은 `computer://` 링크, CLI 등은 절대 경로 텍스트.

## 문제 발생 시

증상별 대응은 `references/troubleshooting.md` 참조. 핵심: ① 본문은 `data-node-id`/`data-viewer-type="header"` 형제 수집으로 읽고 **2벌 렌더는 node-id로 dedup**, ② `navigate`/클릭과 `javascript_tool`은 분리 호출, ③ 결과 JSON에 raw URL·쿼리스트링·검색 키워드를 담지 말 것(`[BLOCKED]` 회피), ④ "주석·실무서"는 **패널 탭**을 누를 것(좌측 내비 링크 아님).

## 금지 사항

- 본문에서 확인되지 않은 법리를 지어내지 마세요. 모든 진술은 visit한 주석서·실무서 본문에서 추적 가능해야 합니다.
- 검색어만 우연히 일치하고 쟁점이 다른 섹션을 답변에 끼워 노이즈를 만들지 마세요.
- 출처 표기(도서명·저자·판본) 없이 본문을 그대로 옮기지 마세요.
- 개편 전의 폐기된 방식(직접 검색 URL `/v2/search/commentary`, 숫자 `statute` 코드)을 사용하지 마세요.
