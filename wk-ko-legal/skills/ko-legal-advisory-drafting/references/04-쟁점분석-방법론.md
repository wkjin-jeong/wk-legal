# 자문의견서 작성 — 쟁점 분석 방법론

자문의뢰서를 받아 쟁점을 도출하고, lbox-case-search로 판례를 보강하며, 결론·리스크의 균형을 잡는 방법론. SKILL.md 4단원·5단원의 보조 가이드.

## 1. 쟁점 추출 — 의뢰서로부터 작업 단위 만들기

### 1.1 질의의 종류 식별

자문의뢰서의 질의는 통상 다음 네 종류로 나뉜다. 종류에 따라 검토 깊이와 산출물 형식이 달라진다.

**(가) 적법성 질의**: "...이 적법한지", "...이 ○○법에 위반되는지". 적용 법령·규정 식별 후 요건 충족 여부 검토.

**(나) 효력 질의**: "...의 효력이 어떠한지", "...이 무효인지·취소 가능한지". 사법상 효력(무효·취소·해제), 감독상 효력(인가취소·과징금) 분리.

**(다) 구조·절차 질의**: "...의 거래구조가 가능한지", "...절차를 어떻게 진행해야 하는지". 절차 단계, 필요 서류, 인허가, 결의·승인.

**(라) 리스크 질의**: "...의 법적 리스크는 무엇인지", "...의 분쟁 가능성은 어떠한지". 리스크 카테고리별(사법·감독·형사·평판) 평가.

한 의뢰서에 여러 종류의 질의가 섞여 있을 수 있다. 종류별로 답변 구조가 달라지므로 분류부터 명확히 한다.

### 1.2 쟁점 트리 그리기

각 질의에 대해 다음 흐름으로 쟁점 트리를 그린다(머릿속 또는 별도 메모).

```
질의: 의뢰인이 X를 할 수 있는가?
├─ 1차 쟁점: X가 ○○법의 적용대상인가?
│   ├─ 부쟁점 1-1: ○○법 제○조의 정의에 X가 포함되는가?
│   ├─ 부쟁점 1-2: 적용 예외(제○조 단서)에 해당하는가?
│   └─ 부쟁점 1-3: 시점상 ○○법이 적용되는가? (개정 전후)
├─ 2차 쟁점: X에 대해 ○○법의 어떤 의무가 부과되는가?
│   ├─ 부쟁점 2-1: 신고·등록 의무
│   ├─ 부쟁점 2-2: 약관 명시·설명 의무
│   └─ 부쟁점 2-3: 자금세탁방지 의무
├─ 3차 쟁점: 의뢰인이 위 의무를 이행할 수 있는가?
└─ 4차 쟁점: 위반 시 효과는?
    ├─ 부쟁점 4-1: 사법상 효력
    ├─ 부쟁점 4-2: 감독상 제재
    └─ 부쟁점 4-3: 형사책임
```

쟁점 트리를 그리면 **결론을 가르는 핵심 변수**가 어디서 분기하는지 보인다. 결론이 갈리는 부쟁점은 검토 의견 본문에 별도 항으로 분리한다.

### 1.3 결론을 가르는 변수의 식별

자문의견서는 변호사가 결론에 자신감을 가지고 의뢰인에게 제출하는 문서이지만, 사실관계의 일부 변수가 결론을 가른다면 그 변수를 명시적으로 노출해야 한다. 결론을 가르는 변수의 예:

- 거래상대방의 법적 지위(자본시장법 — 전문투자자/일반투자자, 금소법 — 일반금융소비자/전문금융소비자).
- 의무 이행의 시점(신고·등록·약관 변경 통지의 시점).
- 의사표시의 명의자·도달 여부.
- 거래의 목적·동기(부당이득의 법률상 원인 유무, 주식 양도제한의 적용범위).
- 분쟁의 사실관계 변수(누가 먼저 의무 이행을 거절했는지, 손해 발생의 인과관계).

이 변수가 자문의뢰서에서 확정되지 않았다면 두 갈래로 나누어 검토하거나(가정 조건 명시), 사용자에게 짧게 되묻는다.

## 2. lbox-case-search 활용 — 쟁점별 판례 보강

### 2.1 검색 호출의 단위

판례 검색은 **쟁점 단위**로 호출한다. 다음 원칙을 지킨다.

- 한 쟁점 = 한 검색어 (또는 동의어·상위어를 활용한 1~3차 검색).
- 한 호출에 모든 쟁점을 묶지 않는다. 검색 정확도가 떨어진다.
- 검색어는 핵심 법률개념·키워드로 짧게 잡는다(예: "전자금융거래 이용자 중과실", "주식양도제한 정관 효력").

### 2.2 검색어 선정 가이드

쟁점에서 검색어를 뽑을 때 다음 패턴을 활용한다.

**(가) 법령·조문 키워드**: 핵심 조문이 정해져 있으면 조문이름·조문번호로 검색.
- 예: "전자금융거래법 제9조" → "전자금융거래법 제9조 책임".

**(나) 법률개념 키워드**: 법리가 정해져 있으면 법률개념으로.
- 예: "약관 명시 설명의무 면제사유", "양도금지특약 제3자 효력".

**(다) 거래·사안 키워드**: 사실관계가 닮은 사례형 판례를 찾을 때.
- 예: "보이스피싱 면책", "키코 통화옵션", "지급정지 가처분".

**(라) 행위 키워드**: 특정 행위의 효력·책임을 찾을 때.
- 예: "근저당 변경등기 효력", "주식 명의개서 효력".

검색어가 한 번에 안 잡히면 좁혀가거나 넓혀간다.
- 너무 많이 잡히면(노이즈가 많으면) 동시검색어 추가, `caseType=civil`, `courtType=대법원` 등 한정.
- 너무 적게 잡히면 동의어·상위어로 넓히거나, 사실관계 키워드를 추가.

### 2.3 판례의 본문 인용 — 직접 인용 우선

**대표 판례의 법리 설시 부분은 따옴표로 감싸 원문 그대로 인용**한다. 길이 제한을 두지 않는다. 한국 대법원 판결문은 저작권법 제7조 제3호에 따라 저작권 보호 대상이 아니다. 요약·재진술은 법리 의미를 미세하게 왜곡할 위험이 있어 자문의견서의 신뢰도를 깎는다.

직접 인용 표기 예:

```
대법원 2018. 7. 12. 선고 2017다235647 판결은 "[원문]"이라고 판시하고 있습니다.
```

같은 취지의 후속 판례를 묶어 표기할 때는:

```
"...라는 것이 대법원의 일관된 입장입니다(대법원 ○○ 판결, 대법원 ○○ 판결 등 참조)."
```

### 2.4 상급심·하급심 흐름의 점검

하급심 판례를 인용할 때는 그 사건의 상급심이 있는지 lbox에서 확인하고, 상급심 결론이 결과를 뒤집었으면 상급심 결론을 우선 인용한다. 자문의견서에 하급심만 인용하고 상급심을 누락하면 잘못된 결론을 도출할 위험이 크다.

### 2.5 직접 사례 부족 시

자문 사안의 정확한 쟁점을 다룬 판례가 부족할 때:

1. lbox 검색어를 변경해 1~2차 재검색.
2. 그래도 부족하면 유사 분야 법리를 유추 적용. 단 본문에 "직접 사례가 부족하여 ○○ 분야의 법리를 유추 적용한다"는 점을 명시.
3. 의견서 결론에 "본 결론은 직접 판례가 부족한 상태에서 일반 법리에 기초한 것이며, 동일 쟁점에 관한 직접 판례가 발견될 가능성을 배제하지 않습니다"식의 한정 문구를 부기.

## 3. 결론·리스크의 균형 — 견해가 명확한 의견서 작성법

### 3.1 결론 어조의 표준 표현

| 강도 | 표현 예 |
|---|---|
| 확신 | "...임이 분명합니다", "...라 할 것입니다" |
| 판단 | "...로 해석됩니다", "...로 판단됩니다", "...라고 보아야 할 것입니다" |
| 우세 | "...로 봄이 타당합니다", "...의 견해가 우세합니다" |
| 양립 | "...의 가능성도 있습니다", "다만 ...라는 견해도 가능합니다" |
| 회피 | (사용 금지) "...일 수도, 아닐 수도 있습니다", "사안에 따라 다릅니다" |

자문의견서의 결론은 통상 "확신" 또는 "판단" 강도로 작성. "양립"은 결론을 가르는 사실관계가 미확정인 경우에 한해 사용.

### 3.2 반대 해석·소수설의 처리

반대 해석·소수설·하급심 다른 견해가 있으면 다음 패턴으로 처리한다.

```
이상의 검토에 비추어 이 사건에서는 ○○라고 해석됩니다.

다만 ○○에 대해서는 [반대 견해]가 가능하나, ○○의 측면에서 위 결론이 타당하다고 할 것입니다.
```

반대 견해를 본문에 노출하는 것은 의뢰인에게 "이 결론에는 다툼의 여지가 있다"는 신호를 주는 것이다. 변호사가 의뢰인에게 신뢰를 얻는 핵심 요소이다.

### 3.3 리스크 카테고리별 평가

은행·금융 사안이라면 다음 다섯 카테고리를 점검:

1. **사법상 리스크**: 거래 무효·취소, 손해배상, 채무 미이행.
2. **감독상 리스크**: 영업정지, 인가취소, 과태료, 과징금, 임직원 제재.
3. **형사상 리스크**: 자본시장법·은행법·외국환거래법·특금법 등 형벌 규정.
4. **평판 리스크**: 사회·언론적 파급, 고객 이탈.
5. **회계·세무상 리스크** (필요 시).

일반 민·상사 사안이라면 사법상 리스크가 중심이고, 형사상 리스크(횡령·배임·사기 등)와 감독상 리스크(공정거래법·하도급법 등)를 부수적으로.

각 리스크는 발생 가능성과 영향도를 분리해 평가하면 의뢰인이 의사결정에 활용하기 편하다.

```
감독상 리스크: 발생 가능성 중간, 영향도 높음 — 영업정지 3개월 가능성 있음.
사법상 리스크: 발생 가능성 낮음, 영향도 중간 — 거래상대방의 손해배상청구 가능성 제한적.
```

### 3.4 결론을 가르는 변수의 명시

결론이 사실관계 변수에 따라 갈린다면, 다음과 같이 처리한다.

```
가. 거래상대방이 일반금융소비자에 해당하는 경우
    [결론 1]

나. 거래상대방이 전문금융소비자에 해당하는 경우
    [결론 2]
```

또는 가정 조건을 본문에 명시:

```
거래상대방이 자본시장법 시행령 제10조 제1항에 따른 전문투자자에 해당한다는 전제 하에,
이 사건 투자권유에 대해서는 금융소비자보호법 제17조의 적합성 원칙이 적용되지 아니합니다.
다만 ○○ 측 자료에 의하면 거래상대방이 전문투자자로 분류되었는지 명확하지 아니하므로,
의뢰인께서는 이 점을 사전에 확인하시기 바랍니다.
```

## 4. 사안의 적용 단락 — 가장 중요한 부분

자문의견서의 핵심은 **"관련 법령·법리"가 아니라 "사안의 적용"** 단락이다. 의뢰인이 알고 싶은 것은 일반 법리가 아니라, 그 법리가 자기 사안에 어떻게 적용되는지이다.

### 4.1 사안의 적용 작성 패턴

**(가) 사실 매핑**: 법리의 요건사실을 사안의 사실관계에 일대일 매핑.

```
[법리 단락에서 식별된 요건]
은행이 약관규제법 제3조에 따라 약관 명시·설명의무를 이행했는지는 (1) 약관 사본의 교부, (2) 중요 내용의 설명, (3) 고객의 이해 확보 여부에 따라 판단됩니다.

[사안의 적용]
이 사건에서 ○○은행은 (1) 비대면으로 모바일 앱을 통해 약관 사본을 제공하였고, (2) 중요 내용에 대해 동영상 설명을 제공하였으며, (3) 고객의 동의 클릭을 받았습니다. 이 중 (2) 동영상 설명만으로 약관규제법 제3조 제3항의 설명의무가 이행되었다고 볼 수 있는지 여부가 핵심 쟁점입니다.
```

**(나) 결론을 가르는 변수의 분리**: 사안의 사실관계 중 결론을 가르는 변수가 있으면 별도 항으로 분리.

```
다만, 이 사건의 경우 동영상 설명에 약관 제○조 ○항(○○ 면책)에 대한 별도 안내가 포함되었는지에 따라 결론이 달라질 수 있습니다.

가. 동영상 설명에 약관 제○조에 대한 별도 안내가 포함된 경우
    [결론]

나. 동영상 설명에 약관 제○조에 대한 별도 안내가 포함되지 않은 경우
    [결론]
```

**(다) 가정 조건 명시**: 자문의뢰서에서 확인되지 않는 사실에 대해 가정을 두는 경우 명시.

```
의뢰인께서 제공하신 사실관계에 의하면 의뢰인 측 직원의 비밀번호 관리에 중과실이 없었던 것으로 보이나, 이 사실은 향후 분쟁에서 다투어질 수 있는 사항입니다. 본 의견은 의뢰인의 진술이 사실로 인정된다는 전제 하에 작성된 것입니다.
```

### 4.2 사안의 적용에서 피해야 할 함정

- **법리 단락의 반복**: 사안의 적용에서 법리를 또 풀어 쓰면 분량만 늘고 분석이 흐려진다. 법리 단락에서 정리한 요건만 인용하고 곧바로 사실에 매핑.
- **추측에 의한 사실 보충**: 자문의뢰서에 없는 사실을 추측으로 채우면 위험하다. 모르면 가정 조건으로 명시하거나 사용자에게 확인.
- **사실과 평가의 혼재**: "○○가 ○○하였으므로 ○○로 판단됩니다"식 한 문장에 사실과 평가가 섞이면 가독성이 떨어진다. 사실 진술 → 평가 순서로 분리.

## 5. 검토 의견의 분량 가이드

### 5.1 단순 사안 (질의 1~2개, 사실관계 단순)

- 쟁점: 1~2 문장.
- 관련 법령·법리: 1~2 단락. 핵심 조문 인용 + 대법원 판례 1건 직접 인용.
- 사안의 적용: 1~2 단락.
- 결론: 2~3 문장.

질의별 본문 약 30~80줄.

### 5.2 중간 정도 (질의 3~5개)

- 쟁점: 1~3 문장.
- 관련 법령·법리: 2~3 단락. 핵심 판례 1~3건 직접 인용 + 후속 판례 묶음.
- 사안의 적용: 2~4 단락. 결론을 가르는 변수가 있으면 별도 항으로 분리.
- 결론: 3~5 문장. 반대 해석·리스크 부기.

질의별 본문 약 80~200줄.

### 5.3 복잡한 사안 (질의 다수, 외국법 검토, 다수 판례 분석)

- 쟁점: 트리 형태로 부쟁점까지 정리.
- 관련 법령·법리: 부쟁점별로 별도 항. 학설·하급심 다른 견해 함께 정리.
- 사안의 적용: 부쟁점별로 별도 항. 변수별 결론 분리.
- 결론: 5문장 이상. 리스크 평가 별도 항.

질의별 본문 약 200~500줄.

## 6. 산출물 점검 체크리스트

초안 완성 후 의견서 전체에 대해 다음을 점검한다.

### 6.1 형식 점검

- [ ] 표제(수신·발신·일자·제목·문서번호) 작성.
- [ ] 사안의 개요가 시간순으로 정리되어 있다.
- [ ] 질의사항이 자문의뢰서의 질의와 일치한다 (누락·왜곡 없음).
- [ ] 검토 전제·한정사항이 명시되어 있다.
- [ ] 검토 의견이 질의별로 4단 구조(쟁점·법리·적용·결론)를 따른다.
- [ ] 결론 요약이 첫 페이지 또는 말미에 있다.
- [ ] 항·호·목 분류가 "1. → 가. → (1) → (가)" 한 가지로 통일되어 있다.

### 6.2 내용 점검

- [ ] 각 질의에 대해 결론이 명확하다 (회피적 결론 없음).
- [ ] 결론을 가르는 사실관계 변수가 식별되어 본문에 노출되어 있다.
- [ ] 적용 법령·약관·감독규정의 조문이 정확히 인용되어 있다 (시행일자 점검).
- [ ] 인용된 판례가 lbox에서 확인된 것이다 (추정 인용 0건).
- [ ] 핵심 판례는 직접 인용(원문 따옴표) 형식이다.
- [ ] 반대 해석·소수설·리스크가 적절히 노출되어 있다.
- [ ] 은행·금융 사안의 경우 감독상 리스크가 별도 항으로 평가되어 있다.
- [ ] 가정 조건이 본문에 명시되어 있다.

### 6.3 문체 점검

- [ ] 격식체("–입니다/–합니다"체)로 통일되어 있다.
- [ ] 의뢰인·거래상대방 호칭이 일관된다.
- [ ] 추측·완곡 표현("...로 보입니다", "...로 사료됩니다")이 남용되지 않았다.
- [ ] 한자 병기 없이 한글로만 표기되어 있다.
- [ ] 영문 약칭은 첫 등장 시 풀어 쓰고 영문 원어 병기되어 있다.
- [ ] 오탈자·중복·누락이 없다.

### 6.4 사용 편의 점검

- [ ] 의뢰인이 결론 요약만 보고도 의사결정의 큰 그림을 잡을 수 있다.
- [ ] 의뢰인이 후속 액션을 취해야 한다면 권고사항이 명시되어 있다.
- [ ] 추가 사실 확인이 필요한 사항이 변호사에게 안내되어 있다.
