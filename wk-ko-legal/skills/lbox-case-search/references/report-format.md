# lbox-case-search 보고서 양식 reference

보고서 작성 직전에 이 파일을 읽고 그대로 따른다.

다음 형식의 markdown 파일을 저장합니다.

```markdown
# 질의: <사용자의 원래 질문>

## 검색 조건

- 검색어: <검색에 사용한 키워드(재검색 시 모두 나열)>
- 사건유형: <민사/형사/행정/가사 또는 미지정>
- 법원: <대법원 또는 미지정>
- 선고기간: <최근 N년 또는 미지정>
- 검색일자: <YYYY-MM-DD>
- 검색결과: <조회한 판례 N건 + 추가로 조회한 관련/상급심 판례 M건 등>

## 정리된 답변

<사용자 질의에 대한 답변. 판례 내용을 종합한 설명을 자연스러운 문단으로 작성.
각 주장/진술 끝에 [대법원 2023다12345] 형식으로 출처 사건번호를 인라인 인용.
하나의 진술이 여러 판례로 뒷받침되면 [대법원 2023다12345; 2022다67890] 형식으로 묶어 표기.
결론을 가르는 사실관계 변수(통지 명의자, 대리권 위임 여부, 송달방식 등)를 명확히 구분해 정리.>

## 참고 판례

### [사건번호] 사건명

- **법원·선고일자·종류**: <예: 대법원 2023. 5. 8. 선고 / 판결(전원합의체 여부)>
- **판시사항**: ...
- **판결요지**: ...
- **관련 내용**: <사용자 질의와의 관련 부분 요약>
- **결론**: <원고승/패, 상급심 흐름 등>
- **원문**: https://lbox.kr/precedent/<법원>/<사건번호>

### [사건번호] 사건명

...

## 참조 대법원 선례 정리 (있으면)

| 선례 | 법리 | 따름 판례 수 |
|---|---|---|
| 대법원 ... | ... | ... |
```

저장 후 사용자에게 결과 파일 경로를 안내합니다.

- **Cowork 등 데스크톱 환경**: `computer://` 링크 사용. 경로는 사용자의 실제 작업 폴더 절대 경로 + 위에서 정한 파일명을 조합.
  - 형식: `[검색 결과 보기](computer://<사용자의 작업 폴더 절대 경로>/<파일명>)`
  - 예: 작업 폴더가 `/Users/홍길동/Claude/Work/`이면 → `[검색 결과 보기](computer:///Users/홍길동/Claude/Work/lbox-search-부당해고-20260629.md)` (실제 사용자별 경로로 치환).
- **그 외 환경(CLI·웹 등)**: 절대 경로 텍스트로 안내하거나 환경이 지원하는 파일 링크 형식을 사용.

**경로 하드코딩 금지**: 위 예시의 `홍길동`·`Documents/Claude/Work` 같은 경로는 사용자 환경마다 다릅니다. 매 답변마다 실제 작업 폴더 경로를 사용하세요.

#### 답변 형식 보강 권장사항

- 결론을 가르는 **사실관계 변수**를 식별하고, 답변에서 이 변수에 따라 결과가 어떻게 달라지는지 정리하세요. 단순 결과 나열보다 사용자가 자기 사안을 매핑하기 쉬워집니다.
- 참고 판례 표기 시 결과(원고승/패)와 패소 사유가 본 쟁점인지 별도 사유인지를 구분해 적으세요. (예: "통지 자체는 인정되었으나 별도 사유(소멸시효 완성)로 원고패")
- 핵심 대법원 선례는 본문 인용과 함께 별도 표로 정리하면 가독성이 좋아집니다. 결과 카드의 **인용 수**(이 판례를 인용한 판례 수)를 "따름 판례 수"로 활용하면 선례의 안정성을 보여줄 수 있습니다.
- **상급심 흐름 표기**: 하급심 판례를 인용할 때 상급심이 있으면 **결론** 항목에 진행 흐름을 명시합니다. 예: "원고일부승(1심) → 파기환송(대법원 ○○다○○○○) → 항소기각(환송 후 2심)". 상급심이 결론을 뒤집은 경우 본문 인용은 상급심 결론을 우선 반영하고 하급심은 "원심/환송 전 판결"로 표기.

#### 직접 사례 부족 시 한계점 표기 형식 (SKILL 5.5단계 참고)

1차·2차 재시도까지 마쳤음에도 직접 사례가 부족한 경우, 다음 **두 위치**에 한계점을 반드시 명시합니다.

**1) 첫머리 — 질의 바로 다음, "검색 조건"과 "정리된 답변" 사이에 별도 알림 박스로**:

```markdown
> ⚠ **검토 한계 안내**: 본 답변은 사용자 질의의 정확한 사실관계를 다룬 직접 판례가 검색 범위에서 충분히 발견되지 않아, 관련 분야의 대법원 법리를 유추 적용하여 작성되었습니다. 결론은 일반 법리를 토대로 한 추정에 가깝고, 동일 쟁점에 관한 직접 판례가 존재할 가능성을 배제하지 않습니다. 자세한 내용은 마지막 "검토 한계" 섹션을 참고하세요.
```

**2) 마지막 — "검토 한계" 섹션**: 다음 항목을 순서대로 기재.

```markdown
## 검토 한계

- **시도한 검색어 (검색 차수별)**:
  - 1차: <검색어>
  - 2차 재시도: <검색어> (변경 사유: <...>)
  - 3차 재시도: <검색어> (변경 사유: <...>)
- **검토한 판례 수**: 페이지 순회 누적 N건, 본문 visit M건, 추가 조회한 대법원·상급심 K건.
- **직접 사례 부족의 이유**: <사실관계가 동떨어진 사례만 발견 / 검색어는 일치하나 쟁점이 다른 사례가 다수 / 핵심 쟁점에 대한 대법원 선례 미발견 등>.
- **추가 검색·자문 제안**:
  - 권장 키워드 후보: <...>
  - 다른 자료원: 대법원 종합법률정보, 국가법령정보센터, 학술 데이터베이스 등.
  - 전문가 자문: 사안의 정확한 결론을 위해서는 해당 분야 전문 변호사 자문 권유.
```

두 위치 모두 명시하는 것이 원칙입니다. 한 곳만 적으면 사용자가 한계를 놓칠 위험이 있습니다.
